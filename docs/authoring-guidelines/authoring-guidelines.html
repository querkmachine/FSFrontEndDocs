Authoring guidelines

Formatting is the most basic means of maintaining uniformity in project code. A misplaced bracket or oddly intended line can turn functional code into an illegible mess, and worse still can lead to your fancy new JavaScript function breaking absolutely everything. It happens to the best of us.

Simply put, we prefer:

  • Four space indentation, no tabs.
  • Uncapped, but sensible line lengths.
  • Meaningful use of whitespace.
  • Multi-line code blocks.
